cookie

cookie 是存储在浏览器的一小段数据， cookie可以同时被前后台操作,cookie可以跨页面存取 cookie是不可以跨服务器访问的、cookie是有生命期的，默认是与浏览器进程相同，如果进程退出cookie会被销毁

cookie特点
  • 以键值对的形式存储，用分号分隔你的cookie
  • path属性指定cookie生效的路径/全站
  • cookie除了值其余的属性是只写
  • 所有的cookie包含他们的属性,所占据的大小不能超过4k并不能无限制的存储
  • 数量是有限制的,浏览器下的cookie不能超过300个
  • 每个服务器不能超过20个,
  • cookie 是不安全的,可以被客户端更改,因此不要用cookie存储重要及敏感的数据
  • php 将所有的cookie存储在$_COOKIE数据里在后台设置cookie使用的是setcookie方法
  • cookie会在第二次请求时生效
封装一个函数setcookie 输入name value path expires 设置一个cookie
//path=/front:表示只在front目录下生效，包含其子目录
//expires设定cookie的生命期,它的值是一个字符串,需要把过期日期转成日期toGMTString()
function setcookie(name,value,path,expires){
        var current_data=new Date();
    //将日期设置一天之后    
    current_data.setDate(current_data.getDate()+expires);
    //将日期转换成GMT字符串
    expires_str=current_data.toGMTString();
    //设置cookie的过期日期
    document.cookie=name+"="+value+";path="+path+";sepires="+expires_str;
    }
    setcookie("uesr","luxps","/",1)

###session

特点
  • session数据存储在服务器端 首先使用session_start开启会话
  • 每一个会话分配一个单独的session_id
  • 该session_id传递到前台，默认的session_id名称是PHPSESSID
  • 前台只能看到session的id，不能修改session的值
  • 使用session之前要开启会话session_start()
  • session存储在session数组$_SESSION
  • session 存储比较安全，但是如果session数量过多会导致服务器性能下降//

加密操作 保证接口的安全性

  • md5
    加密md5 只能单向加密，不能解密
    可以将任意长度字符串编程一个32位的定长的字符串

    $password=”1234556”;
    echo md5($password);

  • sha1
    可以将任意长度字符串编程一个32位的定长的字符串

    $password=”1234556”;
    echo sha1($password);

        //让文字移动

        //设置一个定时器,执行文字移动

            var r=parseInt(Math.random()*256);

            return Math.floor(Math.random() * (max - min) + min);

            div.innerHTML = word;

getdanmu.php

<?php
    //为前台提供全部的弹幕数据
    $fp=fopen("danmu.txt","r");
    $danmu=array();
    while(!feof($fp)){
        $line=fgets($fp);
        if($line){
            $danmu[]=$line;
        }
    }
    fclose($fp);
    echo json_encode($danmu);
?>
